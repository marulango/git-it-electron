Copia un proyecto de GitHub y clonalo localmente

Forks

Hasta ahora has hecho un proyecto local y lo empujaste a GitHub, ¡Pero eso es solo la mitad de la diversión! La otra mitad es trabajar con otra gente y otros proyectos

Cuando tu copias un repositorio, estás creando una copia de él en tu cuenta de GitHub. Tus copia bifurcada empieza su vida como un repositorio remoto — solo existe en tu cuenta de GitHub, no en tu computadora. Forks son usados para crear tu propia versión de un proyecto (está desviación de la original es como tomar un atajo en la carretera) o contrubuir de nuevo en sus cambios (Como correciones de errores o nuevas caraterísticas) añ proyecto final

Para obtener una copia de repositorio de tu cuenta de GitHub en tu computadora tu tienes que clonarla. Esta acción de clonación copia el repositorio remoto en tu computadora así tu puedes trabajar en el localmente.

Bifucar el repositorio Patchwork

El proyecto en el que trabajaremos está aquí github.com/jlord/patchwork. Ve a ese sitio y da click al botón de fork en la parte superior derecha. Una vez que la animación de fork está completa, tienes una copia a tu cuenta. Copia tu HTTP URL de tu fork desde la barra de dirección de tu browser, está es la dirección de tu fork en el servidor de GitHub.

Clona tu Fork Localmente

Ahora, en la terminal, clonarás el repositorio en tu computadora.Eso creará aútomaticamente una nueva carpeta para el repositorio así que no necesitas crear una por ti mismo. ¡Asegúrate de que no lo estés clonando en otra carpeta de repositorio Git!

Asegúrate de que no estás dentro de el directorio 'hello-world' de desafíos anteriores. Para retirarse y dejar esta carpeta y así poder clonar, sigue estos pasos:

Regresa a tu carpeta 'hello-world':
Tip: cd representa cambio de directorio y los dos puntos significan salir de un directorio nivel uno

cd ..

Anora clona:

git clone <URLFROMGITHUB>

Ve a la carpeta que fue creada para tu copia local de la copia del repositorio (en este caso, llamada 'patchwork').

cd patchwork

Ahora tu tienes una copia de repositorio en tu computadora y está automáticamente conectada al repositorio remoto (tu copia bifurcada) en tu cuenta de GitHub.

Conecta tu Repositorio Original

¿Qué pasa si el repositorio original que creaste pasa a cambiar? Querrás ser capaz de atraer en esos cambios también. Así que agreguemos otra conexión remota, esta ocasión de la original, github.com/jlord/patchwork, el repositorio con esta URL.

Puedes nombrar esta conexión remota como quiera, pero típicamente la gente utiliza el nombre de 'upstream', usemos ese nombre para esto.

git remote add upstream https://github.com/jlord/patchwork.git

Para estar seguro de que tienes la configuración de los mandos, escribe git remote -v para enlistar las direcciones almacenadas. Deberás tener un remoto 'original' con la dirección de tu fork's y después un 'upstream' remoto con la dirección del original, con la URL indicada anteriormente en este paso.

Favor de seleccionar tu directorio de repositorio.
  • Add remote connections
  • git remote add <REMOTENAME> <URL>
  • View remote connections
  • git remote -v
