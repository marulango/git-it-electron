Repositorios

Un repositorio es una colección de artículos relacionados. En nuestro casi, con la escritura de software, esta es una colección de archivos dedicados a un proyecto de software. Puedes imaginarlo como un folder de proyecto con todos los archivos relevantes dentro. De hecho, así es como se verá en tu computadora. Muchas veces se le llama “repos” como abreviación.

Tú le dices a Git que es tú proyecto y Git empezará a rastrear todos los cambios a ese folder. Esto lo hace un repositorio Git---- un folder de ítems siendo rastreados por Git. Git rastrea cuando los archivos son agregados, sustraídos o hasta cuando una sola letra del archivo ha sido cambiada. Todo esto además de lo que se hizo y cuando se hace un seguimiento por Git. En software esto es llamado control de versiones..

Usando la Terminal

La terminal (o Bash) es una manera de dar instrucciones a tu computadora para hacer cosas escribiendo comandos preferibles que dar click a aplicaciones con tu mouse. Puedes renombrar archivos, crear folders nuevos, mover entre directorios (folders) y más, todo por ejecutar comandos escritos. Hasta puedes ejecutar un editor de textos (como Vim) en tu terminal ¡Y no tener que irte jamás!

Además de navegar en tu computadora, también puedes usar programas en la terminal que tienen una Interface de Comando de Línea(CLI), que significa que ellos fuejer ejecutarse con comandos en la terminal. Git es uno de estos. Puedes acceder y controlar Git a través de comandos en la terminar, ¡Cómo lo estarás haciendo muy pronto!

En Git-it aprenderás algo sobre acciones básicas de línea de comandos (además de los comandos de Git) los cuales se te describiran conforme sigamos los pasos

Crea un Repositorio

Vas a crear un nuevo folder de un nuevo proyecto y después inicialo como un repositorio de Git. Para hacer las cosas fácil, nombra tu folder con el nombre de tu proyecto, como “hello-world”

¡Abre tu terminal! En la ventana de tu terminal, escribe estos comandos, uno a la vez apretando enter/return después de cada uno.

Primero, haz una nueva carpeta:
Tip: mkdir stands for make directory

mkdir hello-world

Después ve a esa carpeta:
Tip: cd stands for change directory

cd hello-world

Finalmente, dile a Git que inicie (empiece a espiar) esté folder:

git init

El último comando deberá regresar algo empezando con “inicialización vacía de repositorio de Git”. Los otros comandos no regresan nada.

¡Lo hiciste! Si quieres estar más seguro que es un repositorio Git, escribe git status y si no regresa ‘fatal: no es un repositorio Git’ ¡Eres oro!

Favor de seleccionar tu directorio de repositorio.
  • Haz una nueva carpeta (aka haz un directorio)
    • mkdir <FOLDERNAME>
  • Navega en una carpeta existente (aka cambia de directorio)
    • cd <FOLDERNAME>
  • Enlista los archivos de la carpeta
    • ls
  • Activa a Git para una carpeta
    • git init
