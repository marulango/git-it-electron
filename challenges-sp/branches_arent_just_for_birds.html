Agregue una rama a su copia de repositorio para trabajar en sus cambios.

Branches

Los repositorios Git utilizan ramificaciones para aislar el trabajo cuando lo necesita. Es una práctica común cuando se trabaja en un proyecto o con otras personas para crear una branch para mantener los cambios de su trabajo. De esta manera, usted puede hacer su trabajo mientras que la principal llamada comúnmente rama “maestra” queda estable. Cuando el trabajo en su rama está terminado, se fusionan de nuevo en la rama “maestra” principal.

El diagrama de abajo le muestra como ramificar de su rama “maestra”, hacer el trabajo y la combinación cambia de nuevo a “maestro”. Usted puede ramificar de una rama si lo necesita, la rama “maestra” no necesita ser la base.

For a great visualization on how branches work in a project, see this GitHub Guide: guides.github.com/overviews/flow

GitHub Pages

GitHub automáticamente alojará archivos de páginas web estáticas en ramificaciones llamadas ‘gh-pages’. Este servicio gratuito lo llamé GitHub Pages. Dado que el proyecto se crea en una página web que usted copió y editó, está rama es llamada “gh-pages” en vez de “maestra”. Todos los repositorios que tengan una rama ‘gh-pages’ con archivos en página web pueden ser encontrados, en su versión alojada, usando esté diseño de URL:

http://githubusername.github.io/repositoryname

Crea una branch

Cuando creas una rama, Git copia todo lo que está en la rama actual y lo coloca en la rama que se ha solicitado hacerlo.

Mientras todavía está en el interior de tu repositorio local ‘patchwork’, escribe git statuspara ver cuál es tu rama actual. Git responderá que estás en la rama ‘gh-pages’.

Ahora crear una nueva rama y nombrarla "add-<username>", donde 'username' es tu nombre de usuario. Por ejemplo, "add-jlord". Las ramas son sensibles a las mayúsculas y minúsculas, nombra tu rama exactamente en el modo que aparece tu nombre de GitHub..

git branch <BRANCHNAME>

Ahora tienes una rama con un nuevo nombre que es idéntica a ‘gh-pages’.

Para ir a esa rama y trabajar con ella, similar a usar un cd para cambiar el directorio de la terminal, tú checkout una branch. Para ir a tu branch nueva:

git checkout <BRANCHNAME>

Step: Crea un nuevo archivo

Regresando a tu editor de texto:

  • Crea un nuevo archivo llamado"add-<USERNAME>.txt", donde 'username' es tu nombre de usuario. Por ejemplo, "add-jlord.txt".
  • Después, escribe tu nombre de usuario de GitHub en él, es eso y es todo. Por ejemplo, yo escribiría: 'jlord'.
  • Guarda este archivo en el folder de 'contributors' en Patchwork Patchwork/contributors/add-yourusername.txt
  • Después, revisa en tus cambios (Ver abajo).

Check-in

Sigue los pasos para registrar un proyecto:

git status

git add <FILENAME>

git commit -m "commit message"

Ahora push tu actualización a tu fork en GitHub:

git push origin <BRANCHNAME>

Asegurate de que estás pushing al ‘origen’ el cual es la dirección de tu fork en GitHub. Si obtienes una comprobación de error, asegúrate de que tienes tus controles instalados correctamente con el punto ‘origen’ de tu fork y tu punto “upstream” en el repositorio original: git remote -v

Favor de seleccionar tu directorio de repositorio.

El archivo no va en la carpeta de contribuidores

El archive que creaste deberia estar dentro de la carpeta de ‘contribuidores’ en el repositorio Patchwork. Si lo pones en otro lugar, simplemente usa Finder o Windows Explorer para mover tu archivo a la carpeta. Puedes revisar git status otra vez y podrás encontrar lo que se ve en tus cambios. La etapa y luego comete "todo" (-A) de estos cambios (adiciones y supresiones) con las órdenes debajo.

git add -A

git commit -m "move file into contributors folder"

Nombre esperado de la rama: _____

El nombre de la rama deberá coincidir exactamente con tu nombre de usuario. Para cambiar el nombre de tu rama:

git branch -m <NEWBRANCHNAME>

Cuando hagas tus actualizaciones, ¡Verifica de nuevo!

  • Puedes crear y cambiar a una rama en una línea
  • git checkout -b <BRANCHNAME>
  • Crear una nueva branch
  • git branch <BRANCHNAME>
  • Mover a una branch
  • git checkout <BRANCHNAME>
  • Enlistar las branches
  • git branch
  • Renombrar una branch en la que estás actualmente
  • git branch -m <NEWBRANCHNAME>
  • Verificar en cual branch estás trabajando
  • git status
