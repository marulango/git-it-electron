Conecta tu repositorio local y remoto y empuja cambios.

Remotos

Cuando pones algo en GitHub esa copia vive en uno de los servidores de GitHub. Eso lo hace un repositorio remoto porque este no esta en tu computadora, sino en el servidor, “Remoto” y en otro lugar. Empujando tus cambios locales (en tu computadora),te mantienes al día

Otros siempre pueden obtener lo último de tu proyecto atrayendo tus cambios abajo del remoto (y en sus computadoras). Así es como todos pueden trabajar en un proyecto juntos sin necesitar el acceso a tu computadora donde se encuentra almacenada tu copia local.

Crea tu Repositorio Remoto

Quieres sincronizar tu versión local con una almacenada en GitHub.com. Para crear un repositorio remoto, primero crea un nuevo repositorio en GitHub.com.

  • Ve a github.com, inicia sesión y da click en el '+' de la parte superior derecha y después da click en 'Nuevo Repositorio'.
  • Da un nombre que coincida con el nombre de tu repositorio local, “hello-world”, y una pequeña descripción.
  • Hazlo público. Esto significa que estará enlistado en tu perfil público.
  • No inicies con un README porque ya tenemos un archivo, localmente, llamado ‘readme.txt’. Este es una opción de ayuda de GitHub si no habías hecho una.
  • Deja .gitignore y la licencia establece en “ninguno”. No nos preocuparemos de esos en este proyecto.
  • ¡Haz click en Crear Repositorio!

Readmes, .gitignores y Licencias

Estos son archivos comunes en proyectos de código abierto por lo que cuando se crea un Nuevo repositorio en GitHub conseguir un nuevo proyecto es un poco más fácil.

Un readme explica que es el proyecto, cómo se usa, y muchas veces, como contribuir (aunque a veces hay un archivo extra, CONTRIBUTING.md, para esos detalles).

Un .gitignore es una lista de archivos que Git no hará un seguimiento, por ejemplo, ¡Archivos con contraseña!

Un archivo de licencia es el tipo de licencia que tu pones en tu proyecto, la información de los tipos de licencia están aquí: choosealicense.com.

Conectar tu local a tu remoto

Ahora ya conseguiste un repositorio vacío iniciado en GitHub.com. En la parte de arriba verás ‘Quick Setup’, asegúrate que el botón de HTTP está seleccionado y copia la dirección --- esta es la dirección de tu repositorio en el servidor de GitHub.

De regreso en tu terminal, y dentro del folder “hello-world” el cual iniciaste como un repositorio de Git en el desafío anterior, quieres decirle a Git que almacene la dirección de la versión remota en los servidores de GitHub. Puedes tener multiples remotos así que cada uno requiere un nombre. El remoto primario es típicamente llamado origen.

Para agregar un remoto a tu repositorio:

git remote add origin <URLFROMGITHUB>

Tu repositorio local ahora sabe donde está tu repositorio remoto, llamado 'origen', que vive en los servidores de GitHub. Piénsalo como agregar un nombre y una dirección en marcación rápida---- ahora cuando necesites enviar algo ahí, podrás hacerlo.

Usuarios de Windows:

Si tu tienes GitHub de Escritorio en tu computadora, un remoto llamado 'origen' es automaticamente creado en tu repositorio local. En ese caso, sólo deberás de decirle que URL asociar con el origen. Usa este comando en vez de 'añadir':

git remote set-url origin <URLFROMGITHUB>

Push Work to your Remote

Después quieres push (enviar) todo lo que hiciste localmente a GitHub. Esto es algo que harás seguido asi tu versión remoto está al día y coincidiendo con el estado de tu versión local.

Git tiene un sistema ramificado así que tú puedes trabajar en diferentes partes de un proyecto en tiempo diferentes. Vamos a aprender más sobre eso después, pero por default la primera rama es llamada ‘maestra’. Cuando tu empujas (y después atraes) de un proyecto, yo le dices a Git el nombre de la rama que quieres y el nombre del remoto que ahí vive.

En este caso, vamos a enviar nuestra rama llamada ‘maestra’ a nuestro remoto en GitHub llamado ‘origen’.

git push origin master

Ahora ve a GitHub y refresca la página de tu repositorio. ¡WOAH! Ahora todo es lo mismo local y remotamente. ¡Felicidades por tu primer repositorio público!

Favor de selccionar su directorio de repositorio.
  • Añadir conexiones remotas
    • git remote add <REMOTENAME> <URL>
  • Establecer una URL a un remoto
    • git remote set-url <REMOTENAME> <URL>
  • Empijar cambios
    • git pull <REMOTENAME> <BRANCHNAME>
  • Ver direcciones remotas
    • git remote -v
  • Atraer cambios
    • git push <REMOTENAME> <BRANCH>
