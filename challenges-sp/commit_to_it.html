Crea un nuevo archivo en tu nuevo repositorio, agrega algo a ese archivo y confirmar esos cambios con Git.

Commits

Commits son fundamentales para el uso de Git. Esos son los momentos en los cuales tu guardas y describes el trabajo que tu estuviste haciendo. Son los instantes en la línea del tiempo de la historia de tu proyecto.

Crear un nuevo archivo

Ahora que tienes un repositorio empezado vamos a agregar un archivo en él.

Abre tu editor de texto y crea un nuevo archivo vacío. Ahora escribe un poco de texto, quizá escribe “¡Hello!”, y guarda el archivo como “readme.txt” en la carpeta “hello-word” que creaste en el desafío anterior.

Status, Add and Commit Changes

Después revisa el status de tu repositorio para encontrar si ha habido cambios. Tu sabes si has cambiado algo, pero ¿Git lo sabe?

Asegurate de seguir en tu directorio hello-world cuando estés ejecutando estos comandos. Usa Git para ver qué cambio en tu repositorio:

Primero, revisa el status:

git status

Git te dirá que archivo se ha agregado.

Después agrega el archivo que acabas de crear eso se convertirá en una parte de los cambios que vas a commit (aka guardar) con Git:

git add readme.txt

Finalmente, commit esos cambios en el historial del repositorio con un pequeño mensaje que describa las actualizaciones.

git commit -m "Created readme"

Step: Haz Más Cambios

Ahora agrega otra línea a ‘readme.txt’ y guarda el archivo de nuevo.

En la terminal, tu puedes ver la differencia entre el archivo ahora y como era en tu ultima confirmación.

Dile a Git que te muestre la diff:

git diff

Ahora con lo que acabas de aprender arriba, confirma estos últimos cambios.

Favor de seleccionar tu directorio de repositorio
  • Revisar el estado de cambio de un repositorio
  • git status
  • Ver cambios de los archivos
  • git diff
  • Agregar los cambios de un archivo para ser confirmado
  • git add <FILENAME>
  • Agregar todos los cambios de un archivo
  • git add .
  • Para confirmar (aka guardar) los cambios que agregaste con un mensaje corto describiendo los cambios.
  • git commit -m "your commit message"
